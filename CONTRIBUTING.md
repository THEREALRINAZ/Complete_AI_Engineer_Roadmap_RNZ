
# Contributing to the AI Engineer Road-Map

Thanks for wanting to improve this. The goal of this repo is to stay a **clean, foundation-first roadmap** — contributions should make it clearer, not bigger for its own sake.

## Ways to contribute

- **Fix an error** — typo, broken link, outdated tool reference, incorrect table entry.
- **Add a resource** — a course, doc, or repo that teaches a specific layer well (see "Resource guidelines" below).
- **Improve an explanation** — if a concept (RAG, MCP, LangGraph, etc.) could be clearer, open a PR or an issue with your suggestion.
- **Report a gap** — if a whole layer is missing something important, open an issue describing it before submitting a big PR, so we can agree on scope first.

## Resource guidelines

If you're adding a learning resource (course, article, video, repo) to a layer:

- It must be free or clearly marked if it's paid.
- It must still be maintained/accurate — no dead links, no deprecated tools presented as current.
- One-line description max next to the link — no long pitches.
- Prefer primary sources (official docs, original course) over aggregator blogs.

## How to submit a PR

1. Fork the repo.
2. Create a branch: `git checkout -b fix/short-description`.
3. Make your change.
4. Keep the culinary-metaphor voice consistent with the rest of the doc.
5. Open a PR with a short description of what changed and why.

## Reporting issues

Use GitHub Issues for typos, broken links, or suggestions. Please check existing issues first to avoid duplicates.

## Code of conduct

Be respectful. Disagreements about content are fine and expected — keep them focused on the roadmap, not the person.
